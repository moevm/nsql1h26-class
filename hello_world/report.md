Приложение будет представлять собой интерактивную платформу для студентов и администраторов учебных классов. Основная цель — упростить процесс распределения рабочих мест в компьютерных классах.
Визуально итоговый проект будет состоять из:
Карты/списка доступных ПК для класса с текущим статусом(занят, забронирован, свободен, характеристики?)
Панели быстрого бронирования - форма, где студент выбирает временной слот, вводит свои данные и получает подтверждение.
Личный кабинет/Админка: Список всех текущих броней с возможностью фильтрации и редактировании

Стек технологий
База данных: ArangoDB
ArangoDB выбрана в большей степени как универсальный инструмент. ArangoDB позволяет использовать AQL для эффективных JOIN-операций без потери производительности. Например, в MongoDB связи между «Студентом» и «Компьютером» пришлось бы делать например через вложенность (что неудобно при обновлении статуса ПК).
ArangoDB может работать как графовая модель. Например связь «Студент — Бронирование — Место — Аудитория» является графом. ArangoDB позволяет выполнять графовые обходы (например, найти всех студентов, которые сидели за этим ПК за неделю, чтобы выявить виновного в поломке) намного проще, чем реляционные или чисто документные базы.

Backend: Node.js + Express
Разница в итоговой производительности в данном проекте не будет существенной. Поэтому выбор был сделан по личным предпочтениям.

Frontend: Vue.js
Выбор между Vue, React и Angular в данном проекте тоже не критичный.
